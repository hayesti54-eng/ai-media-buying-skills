# Non-Spark / Dark Post Strategy Planner

**Platform:** TikTok Ads
**Category:** launch
**Tier:** pro

## Purpose
Build a complete non-Spark dark post creative strategy for TikTok Ads — covering when to use dark posts instead of Spark Ads, how to maximize native-feel authenticity without organic post backing, creative configuration best practices, and the specific structural advantages of dark posts for testing, iteration velocity, and message control. Dark posts are the primary testing format for high-volume TikTok advertisers and require a distinct strategic framework from Spark Ads.

## When to Use
- Launching new creative concepts that have not yet been organically posted by a creator
- Conducting high-velocity creative testing where new iterations need to launch faster than creator organic posting timelines allow
- Running creatives with specific offer language, compliance copy, or price points that cannot be disclosed publicly on an organic post
- Testing multiple hook variations of the same creative where each variant would not make sense as an independent organic post
- Building a performance creative pipeline that requires message control (specific CTAs, pixel tracking, custom landing pages) that organic posts cannot support

## Inputs Required
- Creative assets: video files in 9:16 format for each dark post variant
- Ad copy (primary text) for each creative
- CTA button type and destination URL per creative
- Campaign objective and conversion event
- Whether this is a brand-new creative launch or an iteration of an existing creative
- Spark Ads baseline data if available (to benchmark dark post performance against)
- Audience targeting configuration
- Any compliance or message control requirements that necessitate dark post over Spark

## What This Skill Does
This skill produces a complete non-Spark dark post launch strategy covering: creative configuration for maximum native feel without organic social proof, optimal ad copy and display name setup, testing architecture for dark post batches, performance benchmarking against Spark Ad equivalents, and the specific optimization techniques that minimize the native-feel deficit that dark posts carry relative to Spark Ads.

## Analysis Workflow
1. Confirm the strategic rationale for dark posts. Map the operator's situation to one of the validated use cases:
   - Speed to market (new iteration needs to launch today, not after organic posting)
   - Message control (specific offer, price, CTA that cannot be public)
   - High-volume testing (5+ creative variants where Spark Auth overhead would bottleneck the pipeline)
   - New creator content not yet organically posted
   - Retargeting creatives with specific messaging not appropriate for organic
2. Configure the creative for maximum native feel without organic social proof:
   - Display name: use the creator's name or a persona name that matches the creative's style (not the brand's corporate name for UGC-style dark posts). The display name is the first impression of authenticity.
   - Profile image: use the creator's face or a relatable persona image, not the brand logo.
   - Ad copy: write in the first-person creator voice, conversational, informal. Mirror the style of organic TikTok captions — short, with a question or curiosity hook. Add relevant hashtags organically.
   - No "Sponsored" override needed — TikTok labels all paid content, but the presentation of the ad identity determines perceived authenticity.
3. Apply native-feel maximization to the creative video itself:
   - The absence of organic social proof on dark posts means the creative must do more work to establish trust. Prioritize: creator credibility signals in the video itself (testimonial phrasing, genuine reaction, specific result claim), strong social proof woven into the script, and imperfect production quality that signals authenticity.
4. Structure the dark post testing architecture:
   - For hook testing: each hook variant should be a separate dark post with identical body and CTA. Name each ad clearly (Creative-[Name]-Hook-A, Hook-B, Hook-C).
   - For format testing: isolate format variables across separate dark post ads within the same ad set or parallel ad sets.
   - For offer testing: separate dark posts allow different offer framings with full message control.
5. Configure URL tracking: since dark posts do not carry organic engagement data, precise UTM and pixel tracking is critical for performance attribution. Verify every dark post URL has complete UTM tagging.
6. Set performance benchmarks accounting for the Spark deficit: dark posts typically underperform Spark Ads on thumbstop and CPM by 10–20%. Adjust CPA expectations accordingly for new dark post tests vs. proven Spark Ad benchmarks.
7. Define the conversion path for dark post winners: if a dark post achieves strong performance, evaluate whether converting it into a Spark Ad (by having the creator post it organically and then authorizing the boost) would improve performance further.
8. Build the dark post launch checklist: creative specs, display name, profile image, ad copy, UTM, pixel event, CTA, destination URL, naming convention.

## Platform-Specific Best Practices
- The biggest native-feel lever for dark posts is the display name and profile image. A dark post that displays as "Sarah | Wellness Coach" with a real person's face performs significantly better than one showing a brand name and logo for UGC-style content.
- Dark posts cannot inherit organic social proof, so the creative itself must generate trust through internal proof signals: specific numbers, visible results, creator credibility framing, or customer count claims within the video.
- Dark posts are the superior format for high-velocity creative testing because they can launch immediately without creator coordination. Speed of testing iteration is a primary competitive advantage in TikTok performance advertising.
- Ad copy (the text above the video) is read by a meaningful percentage of TikTok users before or during the video. Write dark post ad copy as an extension of the creative hook — a single line that adds curiosity without giving away the payoff.
- Dark posts enable full A/B hook testing because each variant is an independent ad with no shared engagement, ensuring clean data separation. Spark Ad hook testing is more complex because it requires the creator to post multiple organic versions.

## Guardrails
- Do not use the brand's corporate logo and name as the dark post display identity for UGC-style creatives — it instantly signals "advertisement" and undermines the creative's native-feel investment.
- Do not launch dark post tests without individual UTM tracking per ad — without it, all dark post performance blends into a single attribution bucket.
- Do not abandon dark posts for Spark Ads simply because Spark benchmarks are higher — dark posts are essential for iteration speed and message control, and both formats have strategic roles in a mature creative pipeline.
- Never run dark posts without verifying pixel event firing on the specific landing page for each ad's destination URL — different dark post variants may point to different landing pages with different pixel configurations.
